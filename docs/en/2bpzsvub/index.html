Overview

The parser defines the start of a section as `--` followed by a single space followed by the name of the section. Each section is responsible for clearing the empty space at its end so that's what the parse sees for the start of the next section.

The first section doesn't have anything in front of it to clear it so empty space could mess with the parser. This feature lets the parser deal with that
