    Environmental Variable Access For Plugins

    Overview

    • []

      Plugins can be granted access to Environmental Variables for use in API calls, etc...

    • []

      Variables are stored in a TOML config file at `~/.config/neopoligen/env.toml``

    • []

      The format of the TOML file is below

    • []

      Variables are passed to plugins via `-- arg`` attributes

    • []

      Every `-- arg`` attribute is pattern matched to replace variable names surrounded by three `$`` on each side with the matching one in the config if the plugin is on the allowed list

    Examples

    Config File Example

    ~/.config/neopoligen/env.toml 

    [env_var_name_goes_here]
token = "the_value_itself"
plugins = ["list", "of", "allowed", "plugin", "names"]

    Content File Examples

    -- some-plugin
-- arg: $$$MY_LOGIN_TOKEN$$$
    -- some-other-plugin
-- arg: --auth-token
-- arg: $$$lowercase_is_fine$$$
